\doxysubsubsection*{Gigabrot\+\_\+\+C++}



This is a C++ implementation of my original Gigabrot project, put into this repo for ease of submission. Its goal is to take in user-\/specified parameters and efficiently produce a \mbox{\hyperlink{class_p_p_m}{PPM}} image render of the \mbox{\hyperlink{class_mandelbrot}{Mandelbrot}} set with stripe-\/average colorization and a pseudo-\/neumorphic normal mapping effect. This project also demonstrates various fundamental aspects of coding using the C++ language. Sample output image shown above. ~\newline

The \href{https://en.wikipedia.org/wiki/Mandelbrot_set}{\texttt{ Mandelbrot set}} is a set of complex numbers named for \href{https://en.wikipedia.org/wiki/Benoit_Mandelbrot}{\texttt{ Benoit B. Mandelbrot}} for which the function f\textsubscript{c}(z) = z\textsuperscript{2} + c does not exist when z is iterated from 0. The implications of this set of numbers range far and wide in the world of mathematics, and it is often a subject of interest in computing due to its many notable properties and ease of computing. It is also widely known for its aesthetic appeal as a fractal and the many coloring techniques that can be implemented.

The stripe average colorization method implemented here, described in Jussi Härkönen\textquotesingle{}s \char`\"{}\+On fractal coloring techniques\char`\"{}, is an extension of the Triangle Inequality Average method and highlights various features of the fractal. It is combined here with a normal vector map calculated to simulate the soft, shadowy nature of \href{https://uxdesign.cc/neumorphism-in-user-interfaces-b47cef3bf3a6}{\texttt{ modern neumorphic UI design}}.

A key aspect of the set\textquotesingle{}s applications in computing is that the iteration is {\itshape embarrassingly parallel}, meaning it can be easily separated into parallel tasks. Thus, the creation of the thread pool in this project. This pool uses only C++ provide libraries and uses an unlocked work queue to allow for job-\/stealing, inspired by \href{https://www.youtube.com/watch?v=zULU6Hhp42w&t=3635s}{\texttt{ Sean Parent\textquotesingle{}s talk}}.\hypertarget{md__d___documents__final__mandelbrot__cpp__final__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{Note\+:}\label{md__d___documents__final__mandelbrot__cpp__final__r_e_a_d_m_e_autotoc_md2}
Parallelization of the main \mbox{\hyperlink{class_mandelbrot}{Mandelbrot}} code has not been implemented. The vector used to cache each row of the set to be written to the \mbox{\hyperlink{class_p_p_m}{PPM}} file stream output throws a wrench in the \char`\"{}embarrassingly parallel\char`\"{} mix and I did not have time to fully debug the corrupted image output. While I removed that part of the code from my {\ttfamily \mbox{\hyperlink{_c_make_c_compiler_id_8c_a0ddf1224851353fc92bfbff6f499fa97}{main()}}}, I have included the test program I used to verify that the thread pool and its exception handling are in fact working.\hypertarget{md__d___documents__final__mandelbrot__cpp__final__r_e_a_d_m_e_autotoc_md3}{}\doxysubsubsection{Built With\+:}\label{md__d___documents__final__mandelbrot__cpp__final__r_e_a_d_m_e_autotoc_md3}
